It turns out using this website project as a motivator for my mom to finally write down some recipes was not as an effective as I thought! However, I will bug her about it over Thanksgiving break. I've added a recipe that I knew personally: the infamous Mabo tofu dish! It will be used for all of my placeholders until more recipes are gathered. The main skeleton of the website is complete.

The grid format, and side by side image layouts are in place from my previous submission. I've since polished up how all the images are displayed; I learned how to overlap titles over images, and added a semi transparent background for contrast.

Also, I made my first ever drop down menu: the Recipes tab! I don't quite have a concrete plan to add more than the "All" page, but the "Lunch", "Dinner" and "Dessert" pages are all there to taunt me into expanding. It sort of there to serve as a filter function of the "All" page. I could either mimic filtering by making identically laid out grid list with different items, or learn how to dynamically load items using javascript.

It still has this raw edge to it that makes it clear it was made with raw code rather than a CMS. More work can be done to make it more personalize but I would like to prioiritize getting the javascript for comments and dynamic filtering hammered in first.

Speaking of which, my interactive contribution is the comment section at the bottom of this recipe page. Following this template, I was able to create event listeners that obtain values from star ratings (radios disguised as stars), a username textbox, and a comment textbox, and outputs status code into into the comment sections. The code also validates if all three fields (rating, username, and comment) are filled, and will give an error if they are not filled. The only issue is that comments are not saved when you leave and come back. I'd like to find a solution for this.
